Cosas a Tener en Cuenta Antes de Subir un Archivo .csv

  1. Formato del Archivo
    • Extensión del Archivo: Asegúrate de que el archivo tenga la extensión .csv.
    • Codificación: El archivo debe estar codificado en UTF-8 para evitar problemas con caracteres especiales.
  2. Estructura y Contenido
    • Encabezados: El primer registro (fila) debe contener los encabezados de las columnas.
    • Separador de Campos: Los campos deben estar separados por comas (,) u otro delimitador especificado.
  3. Formato de Datos
    • Consistencia: Asegúrate de que todos los registros sigan el mismo formato.
    • Fechas: Si el archivo contiene fechas, asegúrate de que estén en el formato esperado.
  4. Datos Requeridos
    • Campos Obligatorios: Verifica que todos los campos obligatorios estén presentes.

Cosas a Tener en Cuenta Antes de Subir un Archivo .xls, .xlsx

  1. Estructura Interna del Documento
    • Encabezados: Asegúrate de que el primer registro (fila) contenga encabezados de columnas claros y que coincidan con los nombres esperados por el sistema.
    • Formato de Celdas: Verifica que las celdas estén en el formato correcto. Por ejemplo, las celdas que deben contener fechas deben estar formateadas como fechas, y las celdas numéricas deben estar formateadas como números.
    • Datos Consistentes: Asegúrate de que todos los datos en cada columna sean del mismo tipo (e.g., solo números en columnas numéricas, solo texto en columnas de texto).
    • Filas y Columnas: Las filas y columnas deben estar bien definidas y no contener celdas vacías innecesarias en lugares críticos.
  2. Datos Requeridos
    • Campos Obligatorios: Verifica que todos los campos obligatorios estén presentes en cada fila. La falta de campos obligatorios puede causar errores durante la importación.
    • Datos Válidos: Asegúrate de que todos los datos sean válidos y estén en el formato esperado. Por ejemplo, si se espera un número de teléfono, verifica que los datos sean números válidos.
  3. Formato de Datos
    • Fechas: Si el archivo contiene fechas, asegúrate de que todas las fechas estén en el formato esperado (e.g., `YYYY-MM-DD`, `MM/DD/YYYY`).
    • Valores Numéricos: Asegúrate de que los valores numéricos no contengan caracteres no numéricos, como comas o símbolos.
  4. Codificación
    • Codificación del Archivo: Aunque los archivos `.xls` y `.xlsx` generalmente usan una codificación binaria propia, asegúrate de que no haya problemas con caracteres especiales que puedan no ser compatibles con el sistema de importación.
  5. Validación y Limpieza
    • Duplicados: Revisa si el archivo contiene registros duplicados que podrían causar problemas en la base de datos o en el sistema de importación.
    • Errores: Busca errores obvios en los datos, como caracteres inválidos, celdas vacías donde no deberían estar, o datos fuera de rango.
